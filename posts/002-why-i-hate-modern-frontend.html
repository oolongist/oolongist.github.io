我为什么讨厌现代前端

2026-03-06


我曾经是个前端开发者。

那时候写网页很简单：HTML 管结构，CSS 管样式，JavaScript 管交互。三样东西，各司其职。打开编辑器，写，保存，刷新，完事。

现在？

现在你要先装 Node.js，然后 npm init，然后选一个框架——React？Vue？Svelte？——接着是路由、状态管理、构建工具、打包优化、CI/CD。写十行代码之前，你要先配置一百行配置文件。

一个个人博客，为什么要用 React？为什么要 hydration？为什么要 SSR？

我见过太多项目：一个简单的展示页面，硬要上 Next.js，结果首屏加载要三秒，JavaScript 报错一堆，SEO 还是烂的。为了"现代化"而现代化，为了"工程化"而工程化。

这个站点用纯 HTML+CSS 手写。没有构建步骤，没有依赖地狱，没有版本冲突。

打开开发者工具，你看不到任何框架的痕迹。就是几个 .html 文件，一个 .css 文件，托管在 GitHub Pages。加载速度快得离谱，因为浏览器不需要解析几 MB 的 JavaScript 才能显示出几个字。

有人说这是"复古"，是"开倒车"。

我不觉得。我觉得这是清醒。大多数网站不需要那么复杂。你的个人博客不需要虚拟 DOM，不需要状态管理，不需要热更新。

你需要的是内容，和让内容能被阅读的方式。

当然，复杂有复杂的场景。大型应用、团队协作、动态数据——这些确实需要工具。但工具是为了解决问题，不是为了制造问题。

当工具本身成为负担，就该停下来问问：我真的需要这个吗？

这个站点的答案是：不需要。
