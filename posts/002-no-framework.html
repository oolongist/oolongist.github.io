写网页不用框架犯法吗

2026-03-06


朋友问我用什么框架做的这个站。

我说没用。

他愣了一下，说"那你怎么做的"。

我说就...写啊。HTML，CSS，没了。

他沉默了很久。


我不知道从什么时候开始，写网页必须先装 Node.js。必须先选一个框架，React Vue Svelte 选一个吧，然后路由状态管理构建优化，配置文件比代码还多。

我就想做几个静态页面。放几个字。几张图。

这也要 npm install 吗？


看过太多项目了。一个展示页面，硬要上 Next.js，首屏三秒白屏，控制台报错一片。问为什么，说"现代前端都这样"。

这样吗。

我觉得不是。


这个站打开很快。因为浏览器不用先跑几 MB 的 JavaScript 才能显示出几个字。

改东西也很快。打开文件，改，保存，刷新。不用等热更新，不用等构建。

有人说这是复古。开倒车。

随便吧。车能开就行。


当然复杂的东西需要复杂工具。我只是觉得，大多数网站没复杂到那个程度。

你的博客不需要虚拟 DOM。真的。
